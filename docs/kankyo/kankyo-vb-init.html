VirtualBox：GNS3 の初期設定¶

セットアップウィザード¶

初めて GNS3 を起動したとき、セットアップウイザードが実行されます。

  1. GNS3 を起動


  2. Next をクリック


  3. Next をクリック


  4. Next をクリック


  5. OK をクリック


    Note

    このエラーは仮想化プラットホームに VMware を使用している場合のものです。今回は Oracle VirtualBox を使用しているので無視して良い。


  6. Virtualization software は VirtualBox を選択


  7. VirtualBox を選択　→　GNS3 VM に割り当てるリソース（ vCPU 数 /メモリサイズ）を指定

    Note

    割り当てるリソースに関して「 Local GNS3 VM Setup Wizard 」に次の記述があります。


    翻訳サイト で翻訳した結果です。

    重要

    GNS3では、最低1つのvCPUコアと2GBのRAMを推奨しています。しかし、多くのデバイスで大規模なトポロジを実行したい場合は、より多くのCPUコアとRAMを推奨します。設定は後からGNS3のGUIを使って調整できます。経験則としては、PCで利用可能なvCPUコアと物理RAMの合計量の半分を割り当てるのが良いでしょう。以下の例では、ラップトップで実行しているため、最小限のリソース割り当てを使用しています。ハイパースレッディングと32GBのRAMを持つi7クアッドコアCPUを持つデスクトップでは、GNS3 VMを4-6個のvCPUと16GBのRAMに割り当てることで、立派なGNS3トポロジを実行することができ、同時にGNS3とWindows（またはLinux）の間でタスクを切り替えるのに十分なリソースを確保することができます。

    ↓


  8. Next をクリック


  9. GNS3 VM の起動待ち


  10. Oracle VirtualBox が起動　→　GNS3 VM が起動　→　OK を押下


  11. GNS3 VM の起動終了


  12. Finish をクリック


  13. GNS3 の起動終了


ターミナルソフトを TeraTerm Pro に変更¶

「 VMware Player：GNS3 のインストール 」で putty をインストールしなかったので、使用するターミナルソフトを TeraTerm に変更します。

  1. Edit をクリック


  2. Preferences… をクリック


  3. Console Applications をクリック


  4. Edit をクリック


  5. Choose a predefined command のプルダウンリストから TeraTerm Pro を選択

    ↓


  6. コマンドを次のように変更

    "C:\Program Files (x86)\teraterm\ttermpro.exe" /W="%d" /T=1 %h %p

  7. OK をクリック


  8. OK をクリック


  9. 変更終了
